## Реализована модель привычки: 

1. Добавлено приложение habits
2. Реализована модель Habit:
    - связь с пользователем (owner)
    - self-связь для связанных привычек (related_habit)
3. Описаны основные поля привычки:
    - место выполнения (place)
    - время выполнения (time)
    - действие (action)
    - признак приятной привычки (is_pleasant)
    - вознаграждение (reward)
    - периодичность выполнения в днях (periodicity)
    - предполагаемая длительность выполнения в секундах (duration)
    - признак публичности (is_public)
4. Добавлены поля аудита (created_at, updated_at)
5. Настроено отображение модели Habit в административной панели Django

Архитектурные решения:

- Модель Habit универсальна и используется как для полезных, так и для приятных привычек
- Бизнес-ограничения и валидаторы не добавлялись на данном этапе и будут реализованы отдельно, в следующей ветке, чтобы не смешивать структуру данных и бизнес-логику
