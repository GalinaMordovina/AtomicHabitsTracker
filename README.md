## Добавлены напоминания о привычках с логированием и периодичностью

В рамках данной ветки реализована система автоматических напоминаний о привычках с использованием Celery + Redis + Telegram. Задача периодически проверяет привычки пользователей и отправляет уведомления в Telegram строго по заданным условиям.

- Настроен Celery worker и Celery Beat
- Подключён Redis как брокер сообщений
- Реализована периодическая задача send_habits_reminders
- Интеграция с Telegram для отправки уведомлений
- Добавлено логирование для удобной отладки и анализа работы задач

Логика напоминаний (задача send_habits_reminders):

- выполняется периодически (через django-celery-beat)
- выбирает только полезные привычки
- проверяет:
    - совпадение текущего времени (по минутам)
    - наличие telegram_chat_id у пользователя
    - соблюдение periodicity (через last_notified_at)
- отправляет Telegram-уведомление владельцу привычки
- защищена от повторных отправок
- устойчива к ошибкам (ошибка одной привычки не ломает всю задачу)

Логирование (используется стандартный logging)
Все ключевые этапы работы задачи логируются:
- старт и завершение
- количество привычек
- отправка уведомлений
- пропуски по времени и периодичности
- ошибки при отправке

Как проверить:
1. Запустить Redis
2. Запустить worker: 
```
python -m celery -A config worker -l INFO -P solo`
```
3. Запустить beat: 
```
python -m celery -A config beat -l INFO`
```
4. Создать привычку: время = текущая минута и пользователь с telegram_chat_id
5. Дождаться уведомления или вручную вызвать:
```
from habits.tasks import send_habits_reminders send_habits_reminders.delay()
```
