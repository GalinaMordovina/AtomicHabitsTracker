## Добавлено CRUD, публичные привычки и пагинация:

1. Реализован API для работы с привычками в соответствии с требованиями ТЗ:

    - Добавлен CRUD для привычек текущего пользователя
    - Реализован отдельный эндпоинт для просмотра публичных привычек
    - Настроена пагинация списка привычек (по 5 элементов на страницу)

2. Эндпоинты

    - GET /api/habits/ - список привычек текущего пользователя (с пагинацией)
    - POST /api/habits/ - создание привычки
    - GET /api/habits/{id}/ - просмотр привычки
    - PUT / PATCH / DELETE /api/habits/{id}/ - редактирование и удаление привычки владельцем
    - GET /api/habits/public/ - список публичных привычек (только чтение)

3. Права доступа

    - Пользователь имеет полный доступ (CRUD) только к своим привычкам
    - Доступ к публичным привычкам доступен всем авторизованным пользователям
    - Попытки изменения или удаления чужих привычек запрещены

4. Архитектурные решения

    - Использованы отдельные ViewSet’ы для пользовательских и публичных привычек
    - Пагинация вынесена в отдельный класс
    - Назначение владельца привычки происходит автоматически при создании

Результат:
- API полностью соответствует требованиям курсового проекта
- Эндпоинты протестированы через Postman
