## Добавлен HabitSerializer с валидацией по ТЗ

1. Добавлен HabitSerializer с валидацией по ТЗ:
    - запрет одновременного указания reward и related_habit
    - ограничение duration ≤ 120 секунд
    - ограничение periodicity ≤ 7 дней
    - related_habit может быть только приятной привычкой
    - у приятной привычки запрещены reward и related_habit
2. Добавлен базовый HabitViewSet для проверки и дальнейшего расширения CRUD
3. Реализовано автоматическое назначение владельца привычки (owner) при создании

Результат:
Валидаторы проверены через Postman, некорректные сценарии возвращают 400 Bad Request с понятными сообщениями.
